此页面会扫描当前登录用户下已经 rsync 上传完成、但缺少 cloud_id 的采集记录， 并重新调度云端登记通知。它不会重复发起 rsync 文件上传，只补偿云端数据登记步骤。

补偿范围

  • 仅处理当前登录用户自己的记录。
  • upload_status 必须为 completed。
  • cloud_id 必须为空，表示还未登记云端。
  • cloud_notify_status 不能是 notifying。
  • 记录必须存在 task_id 和 output_dir。

环境说明

  • 默认登录环境：prod · 192.168.21.139。
  • 可在统一登录弹窗切换到 test · 192.168.21.138。
  • 登录 API 根据环境切换；补偿 API 使用当前边端服务地址。
